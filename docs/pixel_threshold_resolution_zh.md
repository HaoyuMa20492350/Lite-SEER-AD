# 固定 Dice 阈值问题结论

## 能否解决

可以改善，但不能通过测试异常标签或测试掩码选择阈值。后者只会得到
oracle Dice，属于测试集泄漏，不能作为正式论文结果。

当前正式协议保留：

- 从官方训练正常图生成确定性 `blob`/`scratch` 合成缺陷；
- 使用 seed `7, 13, 23`；
- 正常像素 FPR 上限为 `0.5%`；
- 在约束内冻结合成 Dice 最优阈值；
- 测试掩码只用于计算最终 Dice，不参与阈值选择。

## 已验证的替代方案

1. 仅使用训练正常像素 99.5% 分位数：
   33 类宏平均 Dice 为 `0.3008`，低于当前 `0.3330`。
2. 双阈值滞后分割：
   33 类宏平均 Dice 为 `0.3068`，同样低于当前结果。
3. 增加 `spot`/`patch` 并使用完整 DTD 纹理：
   合成留一 seed 指标更稳定，但真实域阈值偏高。在试点中，
   MVTec `capsule` 固定 Dice 从约 `0.272` 降至约 `0.139`，
   因此没有覆盖正式结果。
4. 标准与弱强度双层合成：
   能降低部分阈值，但合成留一 seed Dice下降，未通过稳定性门槛。

## 根因

固定阈值差距有两类：

- 阈值域偏差：例如 `zipper`，Pixel AP 尚可，但合成缺陷比分布中的
  真实缺陷更强，冻结阈值偏高。
- 热图不可分：例如 MPDD `bracket_white`，Pixel AP 约为 `0.0029`。
  缺陷像素与背景响应本身无法区分，任何无泄漏阈值都不能修复。

## 下一步

阈值协议不再继续针对完整测试集调参。后续应使用独立开发类别或新的
验证协议研究以下方向，再一次性冻结到测试集：

- 256 分辨率或原生分辨率特征热图；
- 多层、多尺度特征融合；
- 针对微小缺陷的局部高分辨率分支；
- 更接近真实缺陷面积和强度的合成生成器；
- 在合成留一 seed 和独立开发类别上同时通过后，再升级正式协议。

当前正式结果仍以 `tables/fulltest_paper_package/` 为准。
